# 汕大东海岸校区吃饭抽签器 — 设计文档

## 概述

一个帮助汕头大学东海岸校区学生解决"每天吃什么"问题的网页应用。随机从 26 家食堂店铺中抽取推荐，支持按分类筛选，并配有管理后台供店铺数据维护。

## 技术栈

- **前端**：纯 HTML + CSS + JavaScript（无框架），部署至 Vercel
- **数据库**：Supabase PostgreSQL（免费套餐）
- **认证**：Supabase Auth（邮箱密码登录，仅管理后台需要）
- **部署**：Vercel（连接 GitHub 仓库自动部署）

## 数据库设计

### shops 表

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | int8 | 主键，自增 | PRIMARY KEY, GENERATED ALWAYS AS IDENTITY |
| name | text | 店铺完整名称（如"一饭粿条"）| NOT NULL, UNIQUE |
| canteen | text | 所属食堂（一饭/二饭/三饭/D饭）| NOT NULL |
| categories | text[] | 分类数组（如 ["饭类"]，沙县为 ["饭类","粉面/粿条"]）| NOT NULL, DEFAULT '{}' |
| created_at | timestamptz | 创建时间 | DEFAULT now() |

### categories 表

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | int8 | 主键，自增 | PRIMARY KEY, GENERATED ALWAYS AS IDENTITY |
| name | text | 分类名称 | NOT NULL, UNIQUE |
| created_at | timestamptz | 创建时间 | DEFAULT now() |

### RLS 策略

- **shops 表**：公开可读（SELECT for everyone），仅管理员可写（INSERT/UPDATE/DELETE for authenticated users）
- **categories 表**：同上

## 页面设计

### 抽签页（`/`）

**布局：**
- 顶部：标题栏 "今天吃啥？"
- 筛选区：纵向排列"全部" + 各分类按钮，选中状态高亮
- 结果区：大卡片展示抽签结果（带动画）
- 操作区：大按钮"抽一签！"
- 额外信息：抽中后显示所属食堂

**交互：**
1. 用户选择分类筛选（默认"全部"）
2. 点击"抽一签！"，卡片执行滚动/翻转动画
3. 动画结束显示随机店铺名称 + 所属食堂
4. 支持直接重新抽签

**状态：**
- 加载中：骨架屏或 loading 提示
- 空筛选：所选分类无店铺时的提示
- 正常：展示抽签结果

### 管理后台（`/admin`）

**登录页（未认证时）：**
- 邮箱 + 密码输入框
- 登录按钮

**店铺管理页（已认证）：**
- 顶部：标题"店铺管理" + 退出登录按钮
- 搜索框：实时过滤店铺列表
- 店铺表格：列：名称、食堂、分类、操作（编辑/删除）
- 新增按钮：打开表单弹窗
- 编辑弹窗：修改名称、食堂、分类
- 删除确认：二次确认弹窗

## 部署流程

1. 在 Supabase 中创建项目，执行 `supabase-schema.sql` 建表
2. 插入初始 26 家店铺数据
3. 在 Supabase Auth 中启用邮箱密码认证，创建管理员账号
4. 获取 Supabase URL 和 anon key，配置到前端代码
5. 将代码推送到 GitHub 仓库
6. 在 Vercel 中导入该仓库，自动部署
7. 配置自定义域名（可选）

## 非功能性需求
